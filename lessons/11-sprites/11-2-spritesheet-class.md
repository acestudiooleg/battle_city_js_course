# 📦 Урок 11.2: Клас SpriteSheet

## Завантажуємо спрайт-лист

Створюємо клас який завантажує картинку та дає зручний метод `draw()` для вирізання спрайтів:

```js
// SpriteSheet.js — завантаження та нарізка спрайтів
// Урок 11: NES графіка

/**
 * Приватний клас — завантажує спрайт-лист та надає метод draw()
 * Використовуємо приватний клас + експортований сінглтон,
 * щоб у всій грі був ОДИН екземпляр спрайт-листа
 */
class _SpriteSheet {
    constructor() {
        /** Об'єкт Image — сама картинка */
        this.img = new Image();

        /** Чи картинка вже завантажилась? */
        this.ready = false;

        // Коли картинка завантажиться — ставимо ready = true
        this.img.onload = () => {
            this.ready = true;
            console.log('🖼️ Спрайт-лист завантажено!');
        };

        // Починаємо завантаження
        // '../assets/' — шлях до папки з картинкою
        this.img.src = '../assets/battlecity_general.png';
    }

    /**
     * Малює фрагмент спрайт-листа на Canvas
     * @param {CanvasRenderingContext2D} ctx - контекст Canvas
     * @param {number} sx - X на листі (звідки вирізаємо)
     * @param {number} sy - Y на листі
     * @param {number} sw - ширина вирізки
     * @param {number} sh - висота вирізки
     * @param {number} dx - X на Canvas (куди малюємо)
     * @param {number} dy - Y на Canvas
     * @param {number} dw - ширина на Canvas
     * @param {number} dh - висота на Canvas
     * @returns {boolean} true якщо вдалося намалювати
     */
    draw(ctx, sx, sy, sw, sh, dx, dy, dw, dh) {
        // Якщо картинка ще не завантажилась — нічого не малюємо
        if (!this.ready) return false;

        // Малюємо фрагмент картинки на Canvas
        ctx.drawImage(this.img, sx, sy, sw, sh, dx, dy, dw, dh);
        return true; // вдалося!
    }
}

/** Глобальний сінглтон — один на всю гру */
export const spriteSheet = new _SpriteSheet();
```

### Що таке "сінглтон"?

**Сінглтон** (singleton) — це коли об'єкт створюється **один раз** і використовується всюди. Нам не потрібно 10 копій спрайт-листа — достатньо однієї. Тому ми:

1. Створюємо `_SpriteSheet` з підкресленням (приватний)
2. Створюємо **один** екземпляр `spriteSheet` і експортуємо його
3. Всі файли імпортують `spriteSheet` і використовують той самий об'єкт

## 🗺️ Карта координат спрайтів

Тепер додамо координати всіх спрайтів у цей же файл:

```js
// ─── Координати спрайтів ────────────────────────────────────────

/**
 * Танки: 8 спрайтів 16×16 у рядку.
 * Порядок: UP_F1, UP_F2, LEFT_F1, LEFT_F2, DOWN_F1, DOWN_F2, RIGHT_F1, RIGHT_F2
 *
 * F1, F2 — два кадри анімації гусениць (чергуємо для ефекту руху)
 * Кожен напрямок займає 2 колонки (2 кадри)
 */
export const DIR_COL = {
    up:    0,   // колонки 0-1 (UP_F1, UP_F2)
    left:  2,   // колонки 2-3 (LEFT_F1, LEFT_F2)
    down:  4,   // колонки 4-5 (DOWN_F1, DOWN_F2)
    right: 6,   // колонки 6-7 (RIGHT_F1, RIGHT_F2)
};

// Базові координати танків гравців
export const PLAYER1_SPRITE = { x: 0, y: 0 };    // P1 жовтий
export const PLAYER2_SPRITE = { x: 0, y: 128 };   // P2 зелений

// Базові координати танків ворогів
export const ENEMY_SPRITES = {
    basic: { x: 128, y: 0 },   // сірий (звичайний)
    fast:  { x: 128, y: 32 },  // білий (швидкий)
    power: { x: 128, y: 64 },  // помаранчевий (потужний)
    armor: { x: 128, y: 96 },  // зелений (броньований)
};

// ─── Тайли (стіни) ──────────────────────────────────────────────

/** Тайли — 16×16 блоки на спрайт-листі
 *  Кожен блок розбивається на 4 чвертинки 8×8
 *  (тому що наш TILE = 16, а спрайт = 16, і блок = 2×2 тайли)
 */
export const TILE_SPRITES = {
    brick:    { x: 256, y: 0 },   // цегла
    concrete: { x: 256, y: 16 },  // бетон
};

/** Вода — 2 кадри анімації (чергуємо кожні 500мс) */
export const WATER_SPRITES = [
    { x: 256, y: 32 },  // кадр 1
    { x: 256, y: 48 },  // кадр 2
];

/** Ліс — малюється ПОВЕРХ танків */
export const FOREST_SPRITE = { x: 272, y: 32 };

// ─── Орел (штаб) ────────────────────────────────────────────────

export const EAGLE_SPRITES = {
    alive: { x: 304, y: 32 },  // живий орел
    dead:  { x: 320, y: 32 },  // зруйнований
};

// ─── Spawn-зірка ────────────────────────────────────────────────

/** 4 кадри зірки появи ворога */
export const SPAWN_STAR_SPRITES = [
    { x: 256, y: 96 },
    { x: 272, y: 96 },
    { x: 288, y: 96 },
    { x: 304, y: 96 },
];
```

### Як знайшли координати?

Координати знайдено **піксельним аналізом** файлу `battlecity_general.png` — скриптом що сканує кожну клітинку 16×16 і визначає її вміст за кольорами та формою.
