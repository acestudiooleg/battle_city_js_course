# 4.2: Керування вибухами в Game.js ⚙️

## Що ми будемо робити? 🚀

Сам по собі клас вибуху — це лише креслення. Щоб ми побачили "бабах" на екрані, головний клас гри `Game.js` має навчитися створювати ці об'єкти, зберігати їх у списку, оновлювати в кожному кадрі та вчасно видаляти, коли частинки зникають. Сьогодні ми перетворимо масив `explosions` на динамічну систему ефектів.



## 🔧 Оновлення класу Game.js

Відкрийте файл `Game.js`. Нам потрібно додати імпорт нового класу та налаштувати три головні етапи: створення, оновлення та малювання.

### 📥 Крок 1: Імпорт
Спочатку ми маємо "познайомити" наш головний двигун з класом вибухів. Без цього імпорту спроба створити новий вибух призведе до помилки `ReferenceError`.

```javascript
import { Explosion } from './Explosion.js';
```

### 🛠 Крок 2: Метод створення вибуху
Додамо в клас `Game` спеціальний метод-помічник. Це буде "команда", яку ми зможемо викликати з будь-якої частини програми (наприклад, з менеджера колізій), просто передавши координати та тип вибуху. Метод створює об'єкт та додає його до списку активних ефектів.

```javascript
/**
   * Створення вибуху в заданій позиції
   * @param {number} x - координата X
   * @param {number} y - координата Y
   * @param {string} type - тип вибуху ('small', 'medium', 'large')
   */
  createExplosion(x, y, type = 'small') {
    const explosion = new Explosion(x, y, type, this.logger);
    this.explosions.push(explosion);
  }
```

### 🔄 Крок 3: Оновлення стану (Update)
Вибухи — це тимчасові об'єкти. Якщо ми будемо просто додавати їх у масив і ніколи не видаляти, гра швидко почне "гальмувати". Ми використаємо метод `filter`, щоб кожного кадру залишати лише ті вибухи, які ще активні (`isActive === true`).

```javascript
// Оновлюємо кожний вибух та залишаємо в масиві лише активні
    this.explosions = this.explosions.filter((explosion) => {
      explosion.update(deltaTime);
      return explosion.isExplosionActive();
    });
```


### 🎨 Крок 4: Візуалізація (Render)
Ефекти мають малюватися поверх танків та стін, щоб спалахи та іскри перекривали об'єкти гри. Тому ми додаємо цикл малювання вибухів у самий кінець методу `render`, безпосередньо перед малюванням інтерфейсу.

```javascript
// Малюємо всі активні вибухи поверх танків та поля
    this.explosions.forEach((explosion) => {
      explosion.render(this.ctx);
    });
```

## 🎮 Як це працює?

### 🧹 Самоочищення:
Завдяки перевірці `explosion.isExplosionActive()`, наш масив `explosions` залишається компактним. Як тільки остання частинка вибуху стає прозорою або виходить час його життя, об'єкт видаляється з пам'яті під час наступної перевірки `filter`.

### 🏗️ Централізація:
Метод `createExplosion` стає універсальним інструментом. Тепер нам не важливо, чому стався вибух (влучання кулі, самознищення чи скриптова подія) — ми просто викликаємо цей метод, а все інше двигун зробить автоматично.
