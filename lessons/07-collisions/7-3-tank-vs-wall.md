# 🎮 Урок 7.3: Танк ↔ Стіна — блокування руху

## Танк більше не проїде крізь стіни! 🧱

Час додати `canTankMove()` до `GameField.js`! В уроці 6 танк проїжджав крізь стіни — тепер це закінчиться.

## Додаємо методи до GameField.js

Відкрий `GameField.js` і додай ці методи після `update(dt)`:

---

## Як працює `canTankMove()`

```javascript
canTankMove(tank, nx, ny) {
    // nx, ny — НОВА позиція, куди танк ХОЧЕ переміститися
    // Ми перевіряємо, чи там вільно, і повертаємо true/false

    const w = tank.width;   // ширина танка (32px)
    const h = tank.height;  // висота танка (32px)

    // ── Перевірка 1: межі поля ──
    // || означає "АБО". Якщо ХОЧА Б ОДНА умова true → return false
    // nx < 0 = танк виїхав за лівий край
    // ny < 0 = танк виїхав за верхній край
    // nx + w > FIELD_W = правий край танка виходить за поле
    // ny + h > FIELD_H = нижній край танка виходить за поле
    if (nx < 0 || ny < 0 || nx + w > FIELD_W || ny + h > FIELD_H)
        return false; // за межами → НЕ можна!

    // ── Перевірка 2: стіни (цегла + бетон) ──
    // Проходимо по кожній стіні t у масиві this.walls
    // Якщо прямокутник танка перетинається з прямокутником стіни → НЕ можна
    for (const t of this.walls) {
        // Це формула AABB з уроку 7.1:
        // "лівий край танка лівіше правого краю стіни" І
        // "правий край танка правіше лівого краю стіни" І ... (так для X і Y)
        if (nx < t.x + t.width && nx + w > t.x &&
            ny < t.y + t.height && ny + h > t.y)
            return false; // перетин зі стіною → НЕ можна!
    }

    // ── Перевірка 3: вода ──
    // Танки не можуть проїхати через воду (але кулі — можуть!)
    for (const t of this.water) {
        if (nx < t.x + t.width && nx + w > t.x &&
            ny < t.y + t.height && ny + h > t.y)
            return false; // вода → НЕ можна!
    }

    // ── Перевірка 4: штаб (орел) ──
    // Танки не повинні заїжджати на орла
    const e = this.eagle;
    if (e.alive && nx < e.x + e.width && nx + w > e.x &&
        ny < e.y + e.height && ny + h > e.y)
        return false; // штаб → НЕ можна!

    // Якщо жодна перевірка не спрацювала — шлях вільний!
    return true;
}
```

## Візуально

```
Танк хоче рухатись вправо:

    nx = поточна X + speed

    ┌──────┐      ┌────┐
    │ ТАНК │ →    │СТІН│   ← стіна блокує!
    └──────┘      │  А │
                  └────┘

    canTankMove перевіряє:
    nx < стіна.x + стіна.width?  → так
    nx + танк.width > стіна.x?   → так
    ny < стіна.y + стіна.height? → так
    ny + танк.height > стіна.y?  → так
    → Всі 4 умови = ПЕРЕТИН → return false → танк не рухається!
```

## Підключення в `Tank.move()`

В `main.js` замінюємо `() => true` на справжню перевірку:

```javascript
player.move(dir, dt, (tank, nx, ny) => field.canTankMove(tank, nx, ny));
```

А в `Tank.move()` ця функція викликається перед оновленням позиції:

```javascript
if (canMove(this, nx, ny)) {
    this.x = nx;  // рухаємося тільки якщо дозволено!
    this.y = ny;
}
```

## Підсумок

- ✅ `canTankMove()` перевіряє 4 типи перешкод: межі, стіни, вода, штаб
- ✅ Використовує AABB для кожної перевірки
- ✅ Повертає `false` при першому ж зіткненні
- ✅ Танк рухається тільки якщо `canMove()` повертає `true`
