# 🗺️ Урок 2.5: Координатна система — fieldXY vs screenXY

## Два світи координат! 🌍🌍

У нашій грі є **дві системи координат**, і дуже важливо їх не плутати! Це як різниця між адресою в місті та координатами на глобусі.

---

## 📊 Screen-координати (екранні)

Це координати **на Canvas** — в пікселях від лівого верхнього кута:

```
(0,0) ─────────────────────────── (608,0)
  │  BORDER                          │
  │  (16,16)─────────(432,16)        │
  │    │   ІГРОВЕ     │  SIDEBAR     │
  │    │   ПОЛЕ       │              │
  │    │              │              │
  │  (16,432)────────(432,432)       │
  │                                  │
(0,448) ──────────────────────── (608,448)
```

Точка **(0, 0)** — лівий верхній кут Canvas.
Ігрове поле починається в точці **(16, 16)** — бо зсув на `BORDER`.

---

## 🎮 Field-координати (ігрові)

Це координати **на ігровому полі** — в пікселях від лівого верхнього кута поля:

```
(0,0) ──────────────── (416,0)
  │                       │
  │   Танк на fieldX=64   │
  │   fieldY=128          │
  │       🟡              │
  │                       │
(0,416) ─────────── (416,416)
```

Точка **(0, 0)** — лівий верхній кут **ігрового поля**, а не Canvas!

---

## 🔄 Як конвертувати?

```
screenX = FIELD_X + fieldX     →  screenX = 16 + fieldX
screenY = FIELD_Y + fieldY     →  screenY = 16 + fieldY
```

### Приклад:

Танк стоїть на **field(64, 128)**. Де його малювати на Canvas?

```js
const fieldX = 64;
const fieldY = 128;

const screenX = FIELD_X + fieldX;  // 16 + 64  = 80
const screenY = FIELD_Y + fieldY;  // 16 + 128 = 144

ctx.fillRect(screenX, screenY, TANK_SIZE, TANK_SIZE);
// Малюємо на позиції (80, 144)
```

---

## 🔲 Tile-координати (тайлові)

Є ще третя система — **в тайлах**! Замість пікселів — номер клітинки:

```
tx=0  tx=1  tx=2  ...  tx=25
┌────┬────┬────┬─────┬────┐ ty=0
│    │    │    │     │    │
├────┼────┼────┼─────┼────┤ ty=1
│    │    │    │     │    │
├────┼────┼────┼─────┼────┤ ty=2
│    │ 🟡 │    │     │    │
```

Танк на позиції **(tx=1, ty=2)** означає:

```js
const tx = 1;
const ty = 2;

const fieldX = tx * TILE;   // 1 * 16 = 16 пікселів
const fieldY = ty * TILE;   // 2 * 16 = 32 пікселі

const screenX = FIELD_X + fieldX;  // 16 + 16 = 32
const screenY = FIELD_Y + fieldY;  // 16 + 32 = 48
```

---

## 📋 Таблиця конвертації

| Тайлові (tx, ty) | Field (px) | Screen (px) |
|---|---|---|
| (0, 0) | (0, 0) | (16, 16) |
| (1, 1) | (16, 16) | (32, 32) |
| (12, 24) — Орел! | (192, 384) | (208, 400) |
| (8, 24) — P1 spawn | (128, 384) | (144, 400) |
| (24, 0) — Ворог spawn | (384, 0) | (400, 16) |

---

## 💡 Корисна функція

Запам'ятай цю формулу — вона буде використовуватись СКРІЗЬ у грі:

```js
// Тайлові → Field (пікселі на полі)
fieldX = tx * TILE
fieldY = ty * TILE

// Field → Screen (пікселі на Canvas)
screenX = FIELD_X + fieldX
screenY = FIELD_Y + fieldY

// Тайлові → Screen (одразу)
screenX = FIELD_X + tx * TILE
screenY = FIELD_Y + ty * TILE
```

## 🎯 Домашнє завдання

Спробуй додати в `main.js` малювання кольорових квадратиків на різних тайлових позиціях:

```js
// Цегла на позиції (4, 6)
ctx.fillStyle = brickFull;
ctx.fillRect(FIELD_X + 4 * TILE, FIELD_Y + 6 * TILE, TILE, TILE);

// Вода на позиції (10, 12)
ctx.fillStyle = waterCol;
ctx.fillRect(FIELD_X + 10 * TILE, FIELD_Y + 12 * TILE, TILE * 2, TILE * 2);

// Танк (2×2 тайли) на позиції (8, 24)
ctx.fillStyle = '#f8f858';
ctx.fillRect(FIELD_X + 8 * TILE, FIELD_Y + 24 * TILE, TILE * 2, TILE * 2);
```
