# 📐 Урок 2.2: Створюємо constants.js

## Навіщо потрібен файл констант? 🤔

Уяви: ти будуєш будинок і постійно вимірюєш двері рулеткою. Щоразу виходить 200 см. А якщо один раз записати — і використовувати скрізь? Набагато зручніше!

**Константи** — це числа, які ми записуємо один раз і використовуємо скрізь у грі. Якщо захочемо щось змінити — міняємо тільки в одному місці!

---

## 📁 Крок 1: Створюємо файл

Створи новий файл `constants.js` у папці `lesson2/`:

```
lesson2/
├── index.html
├── styles.css
├── variables.css
├── main.js
└── constants.js    ← НОВИЙ ФАЙЛ!
```

---

## 📝 Крок 2: Пишемо константи

Відкрий `constants.js` і напиши цей код:

```js
// === NES Battle City — Константи гри ===

// 🔲 Базовий тайл — найменша клітинка на полі
export const TILE = 16;

// 🚗 Розмір танка (2×2 тайли = 32 пікселі)
export const TANK_SIZE = TILE * 2;

// 📐 Розміри ігрового поля (26×26 тайлів)
export const COLS = 26;
export const ROWS = 26;
export const FIELD_W = COLS * TILE; // 416 пікселів
export const FIELD_H = ROWS * TILE; // 416 пікселів

// 🖼️ Рамка навколо поля
export const BORDER = 16;

// 📊 Бічна панель (sidebar) справа
export const SIDEBAR_W = 160;

// 🖥️ Повний розмір Canvas
export const CANVAS_W = BORDER + FIELD_W + BORDER + SIDEBAR_W; // 608px
export const CANVAS_H = BORDER + FIELD_H + BORDER;              // 448px

// 📍 Зсув ігрового поля від краю Canvas
export const FIELD_X = BORDER; // 16px
export const FIELD_Y = BORDER; // 16px
```

---

## 🧠 Розбираємо код

### Що таке `export`?

```js
export const TILE = 16;
```

Слово `export` означає "дозволити іншим файлам використовувати цю змінну". Це як поставити книгу на полицю в бібліотеці — інші можуть її взяти!

### Що таке `const`?

```js
const TILE = 16;
```

`const` означає "константа" — значення, яке **ніколи не змінюється**. Тайл завжди 16 пікселів!

### Чому ВЕЛИКИМИ ЛІТЕРАМИ?

У програмістів є традиція: константи пишуть `ВЕЛИКИМИ_ЛІТЕРАМИ`. Так одразу видно, що це значення не змінюється. Це як написати "НЕ ЧІПАТИ!" на важливій коробці.

---

## 🔢 Перевіряємо математику

Давай порахуємо разом:

```
Ширина Canvas:
  BORDER + FIELD_W + BORDER + SIDEBAR_W
= 16    + 416     + 16     + 160
= 608 пікселів ✅

Висота Canvas:
  BORDER + FIELD_H + BORDER
= 16    + 416     + 16
= 448 пікселів ✅
```

Запам'ятай ці числа: **608 × 448** — це розмір нашого ігрового вікна!

---

## 💡 Корисне правило

> Якщо число з'являється в коді більше одного разу — зроби його константою!

Замість `ctx.fillRect(0, 0, 608, 448)` ми напишемо `ctx.fillRect(0, 0, CANVAS_W, CANVAS_H)`. Це набагато зрозуміліше!
