## ✍️ Додаємо render() до Tank.js

Відкрий `Tank.js` та додай після конструктора:

```js
    render(ctx, ox, oy) {
        if (!this.alive) return;

        const sx = Math.round(this.x) + ox;
        const sy = Math.round(this.y) + oy;
        const sz = this.width; // 32

        ctx.save();
        this._drawTreads(ctx, sx, sy, sz);
        ctx.fillStyle = this.color;
        ctx.fillRect(sx + 4, sy + 4, sz - 8, sz - 8);
        this._drawBarrel(ctx, sx, sy, sz);
        ctx.restore();
    }

    _drawTreads(ctx, sx, sy, sz) {
        ctx.fillStyle = darkGray;
        const af = this.animFrame; // 0 або 1

        if (this.direction === 'up' || this.direction === 'down') {
            // Ліва гусениця
            ctx.fillRect(sx,          sy + af * 4,            4, sz / 2 - 2);
            ctx.fillRect(sx,          sy + sz / 2 + (1-af)*4, 4, sz / 2 - 2);
            // Права гусениця
            ctx.fillRect(sx + sz - 4, sy + af * 4,            4, sz / 2 - 2);
            ctx.fillRect(sx + sz - 4, sy + sz / 2 + (1-af)*4, 4, sz / 2 - 2);
        } else {
            // Верхня гусениця
            ctx.fillRect(sx + af * 4,           sy,           sz / 2 - 2, 4);
            ctx.fillRect(sx + sz/2 + (1-af)*4,  sy,           sz / 2 - 2, 4);
            // Нижня гусениця
            ctx.fillRect(sx + af * 4,           sy + sz - 4,  sz / 2 - 2, 4);
            ctx.fillRect(sx + sz/2 + (1-af)*4,  sy + sz - 4,  sz / 2 - 2, 4);
        }
    }
```

## 🔍 Розбираємо render()

### Параметри методу

```js
render(ctx, ox, oy)
```

- **`ctx`** — "олівець" для малювання. Ми передаємо його ззовні, коли викликаємо метод
- **`ox`, `oy`** — зсув ігрового поля від краю Canvas (FIELD_X = 16px, FIELD_Y = 16px)

### Чому потрібні `ox` і `oy`?

Танк зберігає свої координати **всередині ігрового поля** — як адреса в місті. Але Canvas малює від **лівого верхнього кута всього вікна**. Ці дві системи координат не збігаються!

Уяви таку схему:

```
Canvas (0,0) ─── лівий верхній кут вікна
│
├── 16px рамка (BORDER)
│
└── Ігрове поле (0,0) ─── звідси рахує танк
    │
    └── Танк: x=128, y=384
```

Щоб намалювати танк на Canvas, треба додати ці 16px рамки:

```js
const sx = this.x + ox;   // 128 + 16 = 144  ← позиція на Canvas
const sy = this.y + oy;   // 384 + 16 = 400
```

### Чому `Math.round()`?

При русі танк рухається плавно і може мати дробову позицію: `x = 128.7`. Якщо малювати з дробовим числом — пікселі "розмиваються" між двома клітинками. `Math.round()` округлює до цілого **тільки перед малюванням**, не змінюючи справжню позицію танка.

### Чому корпус `sx + 4, sy + 4, sz - 8, sz - 8`?

Гусениці займають 4px по боках. Тому корпус малюється з відступом 4px з кожного боку, і його ширина стає `32 - 4 - 4 = 24px`:

```
0px      4px             28px     32px
│  гусен.│    корпус      │гусен.  │
├────────┼────────────────┼────────┤
```

### Чому `ctx.save()` / `ctx.restore()`?

Canvas запам'ятовує поточний колір (`fillStyle`). Якщо змінити колір для гусениць, він залишиться і для наступних малюнків. `save()` робить "знімок" стану перед нашими змінами, `restore()` повертає його назад після — щоб не "забруднювати" решту малюнку.

## 🔍 Розбираємо `_drawTreads`

`ctx.fillRect(x, y, width, height)` — малює закрашений прямокутник.

Гусениця складається з **двох секцій**, і при `animFrame = 1` вони зсуваються на 4px — ефект обертання!

```
animFrame = 0:    ████    ████
animFrame = 1:        ████    ████
```
