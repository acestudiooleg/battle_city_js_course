## 🤔 Навіщо нам "класи"?

Уяви, що ти будуєш армію танків. У кожного танка є:
- Позиція на полі (x, y)
- Напрямок (вгору / вниз / вліво / вправо)
- Колір (жовтий у гравця, червоний у ворога)
- Здоров'я (скільки пострілів витримає)

Якщо писати окремі змінні для кожного танка — це хаос:

```js
// ❌ Так НЕ треба робити:
let tank1X = 100;
let tank1Y = 200;
let tank1Color = 'yellow';
let tank1Hp = 3;

let tank2X = 300;
let tank2Y = 200;
let tank2Color = 'red';
let tank2Hp = 1;
```

**Клас** — це як **креслення** або **форма для печива** 🍪

Ти один раз описуєш "як виглядає танк" — а потім штампуєш скільки хочеш!

## 📐 Анатомія класу

```js
class Tank {
    // Конструктор — викликається при створенні об'єкта
    constructor(x, y, color) {
        this.x = x;
        this.y = y;
        this.color = color;
    }

    // Метод — функція, яка belongs до класу
    move(dx, dy) {
        this.x += dx;
        this.y += dy;
    }

    draw() {
        // малюємо танк
    }
}

// Створюємо об'єкт — "виліплюємо" з форми
const myTank = new Tank(100, 200, 'yellow');
```

---

## 🔑 Ключові поняття

| Поняття | Що це | Аналогія |
|---------|-------|----------|
| **Клас** | Шаблон/опис | Креслення будинку |
| **Об'єкт** | Конкретний екземпляр | Конкретний будинок |
| **Конструктор** | Функція ініціалізації | Заселення будинку |
| **`this`** | Посилання на себе | "Мій власний" |
| **Метод** | Функція об'єкта | Дія танка |

## 🧪 Спробуй у консолі

Відкрий DevTools (F12) і набери:

```js
class Cat {
    constructor(name) {
        this.name = name;
    }
    meow() {
        console.log(this.name + ' каже: Мяу!');
    }
}

const cat = new Cat('Мурчик');
cat.meow(); // Мурчик каже: Мяу!
```
