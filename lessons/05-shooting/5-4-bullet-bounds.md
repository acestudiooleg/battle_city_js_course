# 🎮 Урок 5.4: Куля за межами поля — деактивація

## Куди зникають кулі? 🤔

Коли куля вилітає за край ігрового поля — вона стає **неактивною** і видаляється з масиву. Якщо цього не зробити, кулі будуть летіти далі нескінченно, витрачаючи пам'ять!

---

## Перевірка меж у `Bullet.update()`

Ми вже додали цю перевірку в уроці 5.1. Подивимось на неї уважніше:

```javascript
update() {
    if (!this.active) return; // неактивну кулю не оновлюємо

    this.x += this.vx; // рухаємо по X
    this.y += this.vy; // рухаємо по Y

    // Перевіряємо, чи куля ще на полі (0..416)
    if (this.x < 0 || this.x > FIELD_W ||
        this.y < 0 || this.y > FIELD_H) {
        this.active = false; // вийшла за межі → деактивуємо
    }
}
```


## Чому не видаляємо одразу?

Може виникнути запитання: "Навіщо `active = false`? Чому не видалити кулю одразу?"

Відповідь: ми ітеруємо по масиву куль у `updateBullets()`. Видаляти елемент **під час ітерації** — це помилка! Натомість:

1. `update()` позначає кулю як `active = false`
2. `filter()` створює новий масив без неактивних куль

```javascript
// ❌ Погано: видаляти під час циклу
for (const b of this.bullets) {
    b.update();
    if (!b.active) this.bullets.splice(i, 1); // ПОМИЛКА!
}

// ✅ Добре: спочатку оновити, потім відфільтрувати
for (const b of this.bullets) b.update();
this.bullets = this.bullets.filter(b => b.active);
```

---

## Скільки куль може бути одночасно?

В оригінальній NES-грі кожен танк може мати **одну** кулю одночасно. Ми поки не обмежуємо кількість (це можна додати пізніше), але кулдаун 400 мс вже обмежує частоту стрільби.

- ✅ Ніколи не видаляємо елементи під час ітерації по масиву
