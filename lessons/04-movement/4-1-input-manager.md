# 🎮 Урок 4.1: Слухаємо клавіатуру — InputManager

## Як комп'ютер дізнається, що ти натиснув клавішу?

Уяви, що ти граєш на піаніно. Коли ти натискаєш клавішу — лунає звук. Коли відпускаєш — звук зникає. Так само працює клавіатура в JavaScript!

Браузер генерує дві **події**:
- `keydown` — клавішу **натиснули** (палець опустився)
- `keyup` — клавішу **відпустили** (палець піднявся)

## Що таке `InputManager`?

В іграх потрібно знати **не тільки** яку клавішу натиснули, а й **чи вона зараз утримується**. Наприклад: гравець тримає `W` — танк їде вгору. Відпустив — танк зупинився.

Для цього ми створимо спеціальний клас **InputManager** — "менеджер вводу".

---

## Створюємо `InputManager.js`

Створи новий файл `InputManager.js`:

```javascript
// InputManager.js — зчитування клавіатури
// Урок 4: Рух танка

/**
 * Клас InputManager — менеджер вводу з клавіатури.
 * Слухає події keydown/keyup і зберігає стан кожної клавіші.
 * Дозволяє дізнатися, які клавіші зараз утримуються.
 */
export class InputManager {
    constructor() {
        /**
         * Стан утримуваних клавіш.
         * Ключ — код клавіші (e.code), значення — true/false.
         * Приклад: { KeyW: true, KeyD: false }
         * @type {Object.<string, boolean>}
         */
        this.held = {};

        /**
         * Обробник натискання клавіші.
         * Стрілкова функція (=>) зберігає контекст this.
         * @param {KeyboardEvent} e — подія клавіатури
         */
        this._onDown = (e) => {
            if (e.repeat) return;  // ігноруємо автоповтор при утриманні

            this.held[e.code] = true; // позначаємо клавішу як натиснуту

            // Забороняємо стандартну поведінку для ігрових клавіш
            // (стрілки прокручують сторінку, пробіл натискає кнопки)
            if (this._isGameKey(e.code)) {
                e.preventDefault();
            }
        };

        /**
         * Обробник відпускання клавіші.
         * @param {KeyboardEvent} e — подія клавіатури
         */
        this._onUp = (e) => {
            this.held[e.code] = false; // позначаємо клавішу як відпущену
        };

        // Підписуємося на глобальні події клавіатури
        document.addEventListener('keydown', this._onDown);
        document.addEventListener('keyup', this._onUp);
    }

    /**
     * Повертає напрямок руху гравця на основі утримуваних клавіш.
     * Пріоритет: W (вгору) > S (вниз) > A (вліво) > D (вправо).
     * @returns {string|null} — 'up', 'down', 'left', 'right' або null
     */
    getMovement() {
        if (this.held['KeyW']) return 'up';    // W = вгору
        if (this.held['KeyS']) return 'down';  // S = вниз
        if (this.held['KeyA']) return 'left';  // A = вліво
        if (this.held['KeyD']) return 'right'; // D = вправо
        return null; // жодна клавіша руху не натиснута
    }

    /**
     * Перевіряє, чи є клавіша ігровою (потрібно заблокувати стандартну дію).
     * @param {string} code — код клавіші (e.code)
     * @returns {boolean}
     */
    _isGameKey(code) {
        return ['KeyW', 'KeyA', 'KeyS', 'KeyD', 'Space'].includes(code);
    }
}
```

## Розбираємо код

### `this.held = {}`

Це об'єкт-словник. Коли натискаємо клавішу `W`, в ньому з'являється:
```javascript
{ KeyW: true }
```
Коли відпускаємо:
```javascript
{ KeyW: false }
```

### `e.code` — код клавіші

Кожна клавіша має свій код. Ось основні:

| Клавіша | `e.code` |
|---------|----------|
| W | `KeyW` |
| A | `KeyA` |
| S | `KeyS` |
| D | `KeyD` |
| Пробіл | `Space` |
| ↑ | `ArrowUp` |
| ← | `ArrowLeft` |

### `e.repeat` — повтор клавіші

Коли ти тримаєш клавішу довго, браузер починає надсилати подію `keydown` знову і знову (як автоповтор при друку тексту). Нам це не потрібно — ми дивимось на `this.held`, тому ігноруємо повтори.

### `e.preventDefault()`

Стрілки зазвичай прокручують сторінку. `e.preventDefault()` каже браузеру: "Не роби стандартну дію! Цю клавішу обробляє гра."

### `getMovement()`

Метод перевіряє клавіші **по пріоритету**: спочатку W, потім S, потім A, потім D. Це означає, що якщо натиснути W і D одночасно — танк поїде **вгору** (бо W перевіряється першим).
