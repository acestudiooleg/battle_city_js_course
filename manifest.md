## 2. Технічні обмеження та Архітектура
* **Стек:** Vanilla JavaScript (ES6+), HTML5 Canvas API. Жодних зовнішніх рушіїв чи бібліотек.
* **Графіка:** Використання Tiled JSON для мап та спрайт-ліста `battlecity_general.png`.
* **Canvas Layout (NES-Канон):**
    * **Ігрове поле:** Квадратна зона, сітка $26 \times 26$ маленьких тайлів (по 16px).
    * **Бічна панель (Sidebar):** Розташована праворуч (відображає кількість ворогів, рівень, життя).
    * **Координати:** Чіткий розподіл між `worldX/Y` (ігрова сітка) та `screenX/Y` (Canvas з урахуванням офсетів та Sidebar).
* **Z-Index:** Специфічні об'єкти, як-от Кущі (ID 20), повинні малюватися ПОВЕРХ танків та снарядів.

---

## 3. Правила Ігрової Логіки (Spawn & Topology)
При побудові карти та програмуванні об'єктів суворо дотримуватися канону:
* **Штаб (Eagle):** Завжди внизу по центру, оточений захисним муром. Поразка при влучанні.
* **Точки спавну гравців:** Р1 — ліворуч від штабу, Р2 — праворуч від штабу.
* **Точки спавну ворогів:** Три точки у верхній частині (лівий кут, центр, правий кут). Вороги з'являються по черзі.
* **Руйнування:** Куля руйнує сегмент (1/4 частина тайла) цегли. Бетон не руйнується звичайними кулями.
