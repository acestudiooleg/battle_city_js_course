## Ігрова логіка

### Гравець
- Стартова позиція: тайл (8, 24) — ліворуч від штабу
- 3 життя, респавн через 2 секунди
- Щит невразливості 2 секунди після спавну (мигає)
- Швидкість: 1.5 px/кадр
- Кулдаун стрільби: 400 мс (v3: зменшено з 1500 мс)

### Вороги
- 20 ворогів у черзі на рівень
- Максимум 4 одночасно на полі
- Спавн кожні 3 секунди
- 3 точки спавну: (0,0), (12,0), (24,0) — циклічно
- Spawn-анімація: мигання 2 секунди перед появою

| Тип    | HP | Швидкість | Кулдаун | Колір       |
|--------|----|-----------|---------|-------------|
| basic  | 1  | 1.0       | 1500 мс | червоний    |
| fast   | 1  | 2.0       | 1500 мс | білий       |
| power  | 1  | 1.0       | 800 мс  | жовтогарячий |
| armor  | 4  | 1.0       | 1500 мс | змінюється з HP |

### AI ворогів
1. Рухається в поточному напрямку
2. Кожні 800–2000 мс — випадковий новий напрямок
3. Якщо застряг — негайна зміна напрямку
4. Стріляє автоматично при готовності кулдауну

### Карта рівня
- Формат: 13×13 блоків (кожен = 2×2 тайли)
- Коди: 0=порожньо, 1=цегла, 2=бетон, 3=вода, 4=ліс

| Матеріал | HP  | Руйнується? |
|----------|-----|-------------|
| Цегла    | 1   | Так (NES: пара тайлів знищується з боку влучання) |
| Бетон    | 99  | Практично ні |
| Вода     | —   | Непрохідна для танків, кулі летять |
| Ліс      | —   | Малюється ПОВЕРХ танків (z-index) |

### Штаб (Орел)
- Позиція: тайл (12, 24) — центр низу поля
- Оточений П-подібною стіною з цегли
- Влучання кулі = Game Over

### Колізії
- Куля ↔ стіна: пошкодження тайла, вибух
- Куля ↔ танк: влучання, вибух
- Куля ↔ штаб: знищення штабу, Game Over
- Куля ↔ куля: взаємне знищення (v3: збільшена зона зіткнення +6px для швидких куль)
- Куля за межами поля: деактивація
- Танк ↔ танк: не проходять одне через одного
- Танк ↔ стіна/вода/штаб: блокування руху

## Спрайтова система (v2)

### Спрайт-лист
- Файл: `/public/assets/battlecity_general.png` (400×256 px, 25×16 клітинок по 16×16)
- `image-rendering: pixelated` на Canvas + `ctx.imageSmoothingEnabled = false` для чіткого піксель-арту
- Fallback: якщо спрайт-лист не завантажився — програмне малювання через `fillRect` (colors.js)

### Карта спрайтів (координати на листі)

**Танки (16×16 кожен, 8 спрайтів у рядку: UP×2, LEFT×2, DOWN×2, RIGHT×2):**

| Танк         | x_base | y     | Колір на листі |
|--------------|--------|-------|----------------|
| Player 1     | 0      | 0     | Жовтий (star 0) |
| Enemy Basic  | 128    | 0     | Сірий          |
| Enemy Fast   | 128    | 32    | Сірий (інша форма) |
| Enemy Power  | 128    | 64    | Сірий (інша форма) |
| Enemy Armor  | 128    | 96    | Сірий (інша форма) |

**Тайли (16×16 блоки, вирізаємо чвертинки 8×8 → масштаб 2× до TILE 16px):**

| Матеріал   | Позиція (x, y) |
|------------|----------------|
| Цегла      | (256, 0)       |
| Бетон      | (256, 16)      |
| Вода кадр 1 | (256, 32)     |
| Вода кадр 2 | (256, 48)     |
| Ліс        | (272, 32)      |
| Лід        | (288, 32)      |

**Інші спрайти:**

| Елемент        | Позиція        | Розмір |
|----------------|----------------|--------|
| Орел (живий)   | (304, 32)      | 16×16 → 32×32 |
| Орел (мертвий) | (320, 32)      | 16×16 → 32×32 |
| Spawn-зірка ×4 | (256–304, 96)  | 16×16 кожен |
| Вибух малий ×3 | (256–288, 112) | 16×16 кожен |
| Вибух великий  | (256, 128)     | 32×32 |

### Рендеринг тайлів
Кожен тайл у грі (16×16 px) = одна чвертинка блоку з рівня (13×13 блоків → 26×26 тайлів).
Чвертинка визначається як `(tx % 2, ty % 2)` і вирізається з 16×16 спрайта:
```
ctx.drawImage(img, spriteX + qx*8, spriteY + qy*8, 8, 8, destX, destY, 16, 16)
```

### Рендеринг танків
NES спрайт 16×16 → Canvas 32×32 (TANK_SIZE). Напрямок і кадр анімації визначають колонку:
```
col = DIR_COL[direction] + animFrame  // 0..7
ctx.drawImage(img, spriteX + col*16, spriteY, 16, 16, dx, dy, 32, 32)
```

### Інструмент налагодження
`sprite-debug.html` — відкриває спрайт-лист з сіткою 16×16/8×8 та показує координати при наведенні.

## Технічні рішення

### Grid-snapping
При зміні напрямку танк вирівнюється по тайловій сітці:
- Горизонтальний рух → snap Y до найближчого TILE
- Вертикальний рух → snap X до найближчого TILE

### Z-index (порядок малювання)
1. Рамка (borderBg) + фон поля (чорний)
2. Вода (спрайт, 2-кадрова анімація кожні 500 мс)
3. Стіни (спрайти цегли та бетону — чвертинки 8×8→16×16)
4. Штаб (Орел — спрайт alive/dead, 16×16→32×32)
5. Танки (спрайти 16×16→32×32) та кулі
6. **Ліс (спрайт поверх танків)** ← окремий виклик `renderForest()`
7. Вибухи (система частинок)
8. Sidebar (програмне малювання)
9. Overlay (пауза / game over)

### Координатна система
- `fx, fy` — координати у просторі поля (0…416)
- `screenX = fx + FIELD_X` — координати на Canvas
- Карта 13×13 блоків розгортається у 26×26 тайлів функцією `buildTileList()`

### Спавн ворогів
- Черга `ENEMY_QUEUE` (20 типів)
- Таймер `ENEMY_SPAWN_INTERVAL` (3 сек)
- Циклічний індекс точки спавну (0→1→2→0→…)
- Обмеження `MAX_ACTIVE_ENEMIES` (4)

- Піксельний аналіз спрайт-листа (400×256) для точного мапінгу координат

### v3 — Геймплей фікси та UI
- Canvas масштаб 2× через CSS `transform: scale(2)` (608×448 → 1216×896 візуально)
- Кулдаун стрільби гравця зменшено: 1500 мс → 400 мс
- Куля ↔ куля: збільшена зона зіткнення на ±6px (щоб швидкі кулі не пролітали одна через одну)
- Sidebar переписано в NES-стилі: іконки ворогів 2 колонки, IP лейбл, прапор з рівнем
- `image-rendering: crisp-edges` додано для кращої піксельної чіткості
- Орел (штаб): знайдено правильну позицію спрайта (304, 32) — підтверджено 97% симетрією та піксельним аналізом. Попередня позиція (256, 64) була помилковою (композит тайлів пошкоджень)

### v4 — NES-стиль руйнування цегли
- Кожен цегляний тайл 16×16 тепер має HP=1 (знищується миттєво)
- Куля руйнує **2 тайли одночасно** з того боку блоку, звідки летить (як в оригіналі NES)
- Блок 32×32 = 4 тайли (TL, TR, BL, BR). Куля ↓ → руйнує TL+TR, куля ← → руйнує TR+BR, і т.д.
- Додано `GameField.destroyBrickPair(tile, bulletDir)` — знаходить партнерський тайл у парі
- Додано `GameField.findWallAt(tx, ty)` — пошук тайла за координатами
- `CollisionManager` тепер передає `bullet.direction` у логіку руйнування
- Бетон як і раніше не руйнується

### v5 — Мультиплеєр (2 гравці)
- **P1:** WASD + Space (жовтий танк, спавн tx:8 — ліворуч від штабу)
- **P2:** Стрілки + Enter (зелений танк, спавн tx:16 — праворуч від штабу)
- `Player.js` приймає `playerNum` (1/2), обирає spawn/sprite/color автоматично
- `InputManager.js` розділено на `getMovementP1/P2()` та `isShootP1/P2()`
- `SpriteSheet.js` — P2 зелений спрайт: `(0, 128)` на листі
- `Game.js` — обидва гравці в update/render/collision/sidebar
- Sidebar показує блоки IP та IIP з іконками та життями обох гравців
- Game Over настає коли обидва гравці втратили всі життя
- Без friendly fire (кулі гравців не шкодять одне одному)

### v6 — Фікси інтро та Game Over
- Інтро музика тепер відтворюється при першому натисканні клавіші/кліку (обхід обмеження autoplay браузера)
- Game Over анімація: текст "GAME OVER" піднімається знизу до центру екрану (NES-стиль)
- При Game Over гра заморожена (танки/кулі стоять), але рендер працює: вибухи догорають, вода анімується, текст рухається
- Додано `_updateGameOverAnim(dt)` — окремий update для стану Game Over

### v7 — Артефакти (Power-ups)
- Додано `PowerUp.js` — клас артефакту з мигаючим рендером
- Додано `POWERUP_SPRITES` у `SpriteSheet.js` — 6 спрайтів з листа (y=112, x=256–336)
- 6 типів артефактів:
  - **tank** (336,112) — +1 життя гравцю
  - **shovel** (288,112) — бетон навколо штабу на 20 сек
  - **helmet** (256,112) — щит невразливості на 10 сек
  - **star** (304,112) — підвищення рангу (швидкість + кулдаун + спрайт), 3 рівні
  - **grenade** (320,112) — знищує всіх ворогів на полі
  - **timer** (272,112) — заморожує ворогів на 10 сек
- Спавн: перший через 10–20 сек, наступні через 15–30 сек, один на полі
- Позиція: випадкова, вирівняна по сітці, у вільному просторі
- `GameField.fortifyEagle()` / `unfortifyEagle()` — тимчасовий бетон для лопати
- `Player.activateHelmet()` / `upgradeRank()` — ефекти каски та зірки
- `Game.freezeTimer` — заморозка ворогів (не оновлюються поки таймер > 0)
